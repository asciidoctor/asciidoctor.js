Positioning the TOC without body classes

The html5 backend stamps toc2 toc-left classes on <body> because its stylesheet cannot find the TOC otherwise. The semantic backend emits the TOC as a plain <nav id="toc">, first child of <main> — already addressable. Every frame below embeds the same converted document, byte for byte; only a small stylesheet changes. The fixed-panel variant shows body:has(#toc) doing precisely the job the body class used to do.

<body class="article">
<header>…</header>
<main>
<nav id="toc" class="toc">
<strong class="title" id="toctitle">Contents</strong>
<ol class="sect-level-1">…</ol>
</nav>
…
</main>
</body>

Generated with backend: 'semantic-html5', standalone: true, :toc: — identical in all five frames.

In flow (default)

The TOC stays where the converter put it: first element of <main>. Two rules make it a quiet box.

The variant's entire CSS 
#toc {
  background: var(--toc-bg);
  border: 1px solid var(--line);
  border-radius: 8px;
  padding: 1rem 1.25rem;
  margin: 1.5rem 0;
}

Left rail

A grid on <main>: the TOC takes the first column, every other child flows in the second. No body class — the converter output is untouched.

The variant's entire CSS 
main {
  display: grid;
  grid-template-columns: 13rem minmax(0, 1fr);
  gap: 0 3rem;
}
main > * { grid-column: 2; }
main > #toc { grid-column: 1; grid-row: 1 / 999; }

Right rail

Same grid, columns swapped. Switching sides is one property, not a converter re-run with a different toc-position attribute.

The variant's entire CSS 
main {
  display: grid;
  grid-template-columns: minmax(0, 1fr) 13rem;
  gap: 0 3rem;
}
main > * { grid-column: 1; }
main > #toc { grid-column: 2; grid-row: 1 / 999; }

Sticky rail

The left rail plus position: sticky — the TOC follows the reader. Scroll inside the frame to see it hold.

The variant's entire CSS 
main {
  display: grid;
  grid-template-columns: 13rem minmax(0, 1fr);
  gap: 0 3rem;
}
main > * { grid-column: 2; }
main > #toc {
  grid-column: 1; grid-row: 1 / 999;
  position: sticky; top: 1rem;
  align-self: start;
  max-height: calc(100vh - 2rem);
  overflow-y: auto;
}

Fixed panel (classic toc2)

The full-height fixed sidebar the html5 backend needs body.toc2.toc-left for. body:has(#toc) applies the page offset only when a TOC exists — :has() is exactly what replaces the body class.

The variant's entire CSS 
body:has(#toc) { padding-left: 16rem; }
#toc {
  position: fixed; inset: 0 auto 0 0;
  width: 15rem;
  padding: 1.5rem 1.25rem;
  overflow-y: auto;
  background: var(--toc-bg);
  border-right: 1px solid var(--line);
}

Each frame is an <iframe srcdoc> holding the converter output with one injected <style> (shared typography + the variant rules shown under the frame). Companion proof for admonitions: devdocs/semantic-html5-admonitions.html. Rationale: “Deliberate omissions” in devdocs/semantic-html5-converter.adoc.
